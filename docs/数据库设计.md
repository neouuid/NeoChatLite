## 表结构详细定义

### 1. users (用户表)

存储用户基本信息

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | UUID | PK | 用户 ID |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 用户名 |
| email | VARCHAR(100) | UNIQUE | 邮箱 |
| phone | VARCHAR(20) | UNIQUE | 手机号 |
| password | VARCHAR(255) | NOT NULL | 密码 (bcrypt 哈希) |
| nickname | VARCHAR(50) | | 昵称 |
| avatar | VARCHAR(500) | | 头像 URL |
| bio | VARCHAR(500) | | 个人简介 |
| status | VARCHAR(20) | DEFAULT 'online' | 状态: online/offline/away/busy |
| created_at | TIMESTAMPTZ | DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMPTZ | DEFAULT NOW() | 更新时间 |
| deleted_at | TIMESTAMPTZ | | 删除时间 (软删除) |

#### 索引

```sql
CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_phone ON users(phone);
CREATE INDEX idx_users_status ON users(status);
CREATE INDEX idx_users_deleted_at ON users(deleted_at);
```

### 2. friends (好友关系表)

存储好友关系和好友请求

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | UUID | PK | 关系 ID |
| user_id | UUID | FK, NOT NULL | 用户 ID |
| friend_id | UUID | FK, NOT NULL | 好友 ID |
| alias | VARCHAR(50) | | 好友备注 |
| status | VARCHAR(20) | DEFAULT 'pending' | 状态: pending/accepted/blocked |
| created_at | TIMESTAMPTZ | DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMPTZ | DEFAULT NOW() | 更新时间 |
| deleted_at | TIMESTAMPTZ | | 删除时间 (软删除) |

#### 索引

```sql
CREATE UNIQUE INDEX idx_friends_user_friend ON friends(user_id, friend_id);
CREATE INDEX idx_friends_user_id ON friends(user_id);
CREATE INDEX idx_friends_friend_id ON friends(friend_id);
CREATE INDEX idx_friends_status ON friends(status);
CREATE INDEX idx_friends_deleted_at ON friends(deleted_at);
```

```sql

CREATE INDEX idx_conv_members_role ON conversation_members(role);
```

### 7. messages (消息表)

存储所有聊天消息

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | UUID | PK | 消息 ID |
| conversation_id | UUID | FK, NOT NULL | 会话 ID |
| sender_id | UUID | FK, NOT NULL | 发送者 ID |

| created_at | TIMESTAMPTZ | DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMPTZ | DEFAULT NOW() | 更新时间 |
| deleted_at | TIMESTAMPTZ | | 删除时间 (软删除) |

#### 索引

```sql
CREATE INDEX idx_messages_conversation_id ON messages(conversation_id);
CREATE INDEX idx_messages_sender_id ON messages(sender_id);
CREATE INDEX idx_messages_created_at ON messages(created_at DESC);
CREATE INDEX idx_messages_type ON messages(type);
CREATE INDEX idx_messages_reply_to_id ON messages(reply_to_id);
CREATE INDEX idx_messages_deleted_at ON messages(deleted_at);

```

## 索引设计

### 索引原则

1. **主键索引**: 所有表都使用 UUID 作为主键
2. **唯一索引**: 保证数据唯一性 (用户名、邮箱、手机号等)
3. **外键索引**: 加快关联查询
4. **复合索引**: 优化常用查询模式
5. **软删除索引**: 加速软删除查询

### 常用查询索引优化

| 查询场景 | 索引 |
|---------|------|
| 用户登录 | idx_users_username / idx_users_email / idx_users_phone |
| 获取好友列表 | idx_friends_user_id + idx_friends_status |
| 获取会话列表 | idx_conv_members_user_id + idx_conversations_last_msg_at |
| 获取会话消息 | idx_messages_conv_created |
| 获取未读数 | idx_conv_members_conversation_id + idx_conv_members_user_id |
| 搜索用户 | idx_users_username (gin 索引可选) |

## 数据库配置建议

### PostgreSQL 配置

```sql
-- 连接数
ALTER SYSTEM SET max_connections = 200;

-- 内存配置 (根据服务器内存调整)
ALTER SYSTEM SET shared_buffers = '4GB';
ALTER SYSTEM SET effective_cache_size = '12GB';
ALTER SYSTEM SET work_mem = '64MB';
ALTER SYSTEM SET maintenance_work_mem = '1GB';

-- WAL 配置
ALTER SYSTEM SET wal_buffers = '16MB';
ALTER SYSTEM SET min_wal_size = '1GB';
ALTER SYSTEM SET max_wal_size = '4GB';

-- 检查点配置
ALTER SYSTEM SET checkpoint_completion_target = 0.9;

-- 统计信息
ALTER SYSTEM SET default_statistics_target = 100;
```

### 备份策略

- **全量备份**: 每天凌晨 2 点
- **增量备份**: 每小时 WAL 归档
- **保留策略**: 保留 7 天全量备份，30 天增量备份
