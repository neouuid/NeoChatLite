## 概述

本文档提供 NeoChat 的完整部署指南，包括开发环境、测试环境和生产环境的部署步骤。

---

## 环境要求

### 服务器要求

- **操作系统**: Linux (推荐 Ubuntu 22.04+ / Debian 12+ / CentOS 8+)
- **CPU**: 2 核以上 (推荐 4 核+)
- **内存**: 4GB 以上 (推荐 8GB+)
- **磁盘**: 20GB 以上 (推荐 SSD)

### 软件依赖

- **Go**: 1.21+ (用于编译后端)
- **Node.js**: 18+ (用于编译前端)
- **PostgreSQL**: 14+ (主数据库)
- **Redis**: 7+ (缓存和会话存储)
- **Nginx**: 1.18+ (反向代理，可选)
- **Docker**: 20.10+ (容器化部署，可选)
- **Docker Compose**: 2.0+ (容器编排，可选)

### 网络要求

- 开放端口:
  - `80` / `443`: HTTP/HTTPS (Web 服务)
  - `8080`: API 网关 (可通过 Nginx 代理)
  - `5432`: PostgreSQL (仅内部访问)
  - `6379`: Redis (仅内部访问)

## 快速部署

### 使用 Docker Compose (推荐)

#### 1. 克隆代码

```bash
git clone https://github.com/neochat/neochat.git
cd neochat
```

#### 2. 创建配置文件

```bash
cd backend
cp configs/config.yaml.example config.yaml
```

编辑 `config.yaml`:

```yaml
server:
  port: 8080
  read_timeout: 30
  write_timeout: 30
  log_level: info
  cors:
    allow_origins:
      - "https://your-domain.com"
      - "http://localhost:3000"
    allow_methods:
      - "GET"
      - "POST"
      - "PUT"
      - "DELETE"
      - "OPTIONS"
    allow_headers:
      - "*"

database:
  host: postgres
  port: 5432
  user: neochat
  password: your_secure_password
  dbname: neochat
  sslmode: disable

redis:
  host: redis
  port: 6379
  password: ""
  db: 0

jwt:
  secret: your_very_secure_jwt_secret_key_change_this
  expiry_hour: 24

storage:
  upload_dir: /app/uploads
  max_file_size: 104857600
  max_image_size: 10485760
  allowed_image_types:
    - "image/jpeg"
    - "image/jpg"
    - "image/png"
    - "image/gif"
    - "image/webp"
  allowed_file_types:
    - "image/jpeg"
    - "image/jpg"
    - "image/png"
    - "image/gif"
    - "image/webp"
    - "application/pdf"
    - "text/plain"
```

#### 3. 创建 Docker Compose 文件

```yaml
version: '3.8'

services:
  postgres:
    image: postgres:15-alpine
    container_name: neochat-postgres
    environment:
      POSTGRES_USER: neochat
      POSTGRES_PASSWORD: your_secure_password
      POSTGRES_DB: neochat
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ./initdb:/docker-entrypoint-initdb.d
    networks:
      - neochat-network
    restart: unless-stopped
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U neochat"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    container_name: neochat-redis
    command: redis-server --appendonly yes
    volumes:
      - redis_data:/data
    networks:
      - neochat-network
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  api-gateway:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: neochat-api
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy
    ports:
      - "8080:8080"
    volumes:
      - ./backend/config.yaml:/app/config.yaml:ro
      - uploads_data:/app/uploads
    networks:
      - neochat-network
    restart: unless-stopped
    environment:
      - GIN_MODE=release

  nginx:
    image: nginx:alpine
    container_name: neochat-nginx
    depends_on:
      - api-gateway
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./nginx/ssl:/etc/nginx/ssl:ro
      - uploads_data:/var/www/uploads:ro
    networks:
      - neochat-network
    restart: unless-stopped

volumes:
  postgres_data:
  redis_data:
  uploads_data:

networks:
  neochat-network:
    driver: bridge
```

#### 4. 启动服务

```bash
docker-compose up -d
```

#### 5. 查看服务状态

```bash
docker-compose ps
```

#### 6. 查看日志

```bash
# 查看所有服务日志
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f api-gateway
docker-compose logs -f postgres
docker-compose logs -f redis
```

#### 7. 停止服务

```bash
docker-compose down

# 停止并删除数据卷（谨慎使用）
docker-compose down -v
```

## 生产环境部署

### 1. 服务器准备

#### 更新系统

```bash
# Ubuntu/Debian
sudo apt update && sudo apt upgrade -y

# CentOS/RHEL
sudo yum update -y
```

#### 创建专用用户

```bash
sudo useradd -r -s /bin/false -d /opt/neochat neochat
```

### 2. 安装 PostgreSQL

#### Ubuntu/Debian

```bash
sudo apt install postgresql postgresql-contrib -y

# 启动并启用
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

#### CentOS/RHEL

```bash
sudo dnf install postgresql-server postgresql-contrib -y

# 初始化数据库
sudo postgresql-setup --initdb

# 启动并启用
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

#### 创建数据库和用户

```bash
sudo -u postgres psql
```

```sql
-- 创建用户
CREATE USER neochat WITH PASSWORD 'your_secure_password';

-- 创建数据库
CREATE DATABASE neochat OWNER neochat;

-- 授予权限
GRANT ALL PRIVILEGES ON DATABASE neochat TO neochat;

-- 退出
\q
```

#### 优化 PostgreSQL 配置

编辑 `/etc/postgresql/15/main/postgresql.conf`:

```conf
# 连接设置
max_connections = 200

# 内存设置
shared_buffers = 256MB
effective_cache_size = 1GB
maintenance_work_mem = 64MB
work_mem = 8MB

# WAL 设置
wal_buffers = 16MB
checkpoint_completion_target = 0.9

# 查询规划
random_page_cost = 1.1
effective_io_concurrency = 200
```

重启 PostgreSQL:

```bash
sudo systemctl restart postgresql
```

### 3. 安装 Redis

#### Ubuntu/Debian

```bash
sudo apt install redis-server -y

# 启动并启用
sudo systemctl start redis
sudo systemctl enable redis
```

#### CentOS/RHEL

```bash
sudo dnf install redis -y

# 启动并启用
sudo systemctl start redis
sudo systemctl enable redis
```

#### 配置 Redis

编辑 `/etc/redis/redis.conf`:

```conf
# 绑定地址
bind 127.0.0.1

# 端口
port 6379

# 密码（可选但推荐）
requirepass your_redis_password

# 持久化
appendonly yes
appendfsync everysec

# 最大内存
maxmemory 512mb
maxmemory-policy allkeys-lru
```

重启 Redis:

```bash
sudo systemctl restart redis
```

### 4. 编译和部署后端

#### 编译后端

```bash
cd backend

# 设置 Go 环境
export GOPROXY=https://goproxy.cn,direct
export GOOS=linux
export GOARCH=amd64

# 编译
go build -o neochat-api ./cmd/api-gateway

# 设置可执行权限
chmod +x neochat-api
```

#### 复制文件到生产服务器

```bash
# 创建目录
sudo mkdir -p /opt/neochat
sudo mkdir -p /opt/neochat/uploads
sudo mkdir -p /opt/neochat/configs

# 复制文件
sudo cp neochat-api /opt/neochat/
sudo cp configs/config.yaml /opt/neochat/config.yaml

# 设置权限
sudo chown -R neochat:neochat /opt/neochat
sudo chmod -R 750 /opt/neochat
```

#### 创建 systemd 服务

编辑 `/etc/systemd/system/neochat-api.service`:

```ini
[Unit]
Description=NeoChat API Gateway
After=network.target postgresql.service redis.service
Requires=postgresql.service redis.service

[Service]
Type=simple
User=neochat
Group=neochat
WorkingDirectory=/opt/neochat
ExecStart=/opt/neochat/neochat-api
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal
Environment="GIN_MODE=release"

# 安全设置
NoNewPrivileges=true
PrivateTmp=true
ProtectSystem=strict
ProtectHome=true
ReadWritePaths=/opt/neochat/uploads

[Install]
WantedBy=multi-user.target
```

#### 启动服务

```bash
# 重载 systemd
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start neochat-api

# 查看状态
sudo systemctl status neochat-api

# 设置开机自启
sudo systemctl enable neochat-api

# 查看日志
sudo journalctl -u neochat-api -f
```

## Docker 部署

### 构建后端镜像

创建 `backend/Dockerfile`:

```dockerfile
# 构建阶段
FROM golang:1.21-alpine AS builder

WORKDIR /app

# 安装依赖
RUN apk add --no-cache git ca-certificates tzdata

# 复制 go.mod 和 go.sum
COPY go.mod go.sum ./
RUN go mod download

# 复制源代码
COPY . .

# 编译
RUN CGO_ENABLED=0 GOOS=linux go build -a -installsuffix cgo -o neochat-api ./cmd/api-gateway

# 运行阶段
FROM alpine:latest

WORKDIR /app

# 安装必要工具
RUN apk --no-cache add ca-certificates tzdata

# 创建非 root 用户
RUN addgroup -g 1000 neochat && \
    adduser -D -h /app -s /bin/false -G neochat -u 1000 neochat

# 从构建阶段复制可执行文件
COPY --from=builder /app/neochat-api .

# 创建上传目录
RUN mkdir -p /app/uploads && \
    chown -R neochat:neochat /app/uploads

# 切换用户
USER neochat

# 暴露端口
EXPOSE 8080

# 健康检查
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD wget --no-verbose --tries=1 --spider http://localhost:8080/health || exit 1

# 启动
CMD ["./neochat-api"]
```

### 构建镜像

```bash
cd backend
docker build -t neochat/api-gateway:latest .
```

### 推送镜像到仓库 (可选)

```bash
# 登录到 Docker Hub
docker login

# 标签镜像
docker tag neochat/api-gateway:latest your-registry/neochat/api-gateway:latest

# 推送镜像
docker push your-registry/neochat/api-gateway:latest
```

## 反向代理配置

### Nginx 配置

创建 `nginx/nginx.conf`:

```nginx
user nginx;
worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;

events {
    worker_connections 1024;
    use epoll;
}

http {
    include       /etc/nginx/mime.types;
    default_type  application/octet-stream;

    log_format  main  '$remote_addr - $remote_user [$time_local] "$request" '
                      '$status $body_bytes_sent "$http_referer" '
                      '"$http_user_agent" "$http_x_forwarded_for"';

    access_log  /var/log/nginx/access.log  main;

    sendfile        on;
    tcp_nopush     on;
    tcp_nodelay    on;
    keepalive_timeout  65;
    types_hash_max_size 2048;
    client_max_body_size 100M;

    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript
               application/x-javascript application/xml application/xml+rss
               application/json application/javascript;

    # 上游服务器
    upstream neochat_api {
        server 127.0.0.1:8080;
        keepalive 32;
    }

    # HTTP 重定向到 HTTPS
    server {
        listen 80;
        server_name your-domain.com;
        
        # Let's Encrypt 验证路径
        location /.well-known/acme-challenge/ {
            root /var/www/html;
        }
        
        location / {
            return 301 https://$server_name$request_uri;
        }
    }

    # HTTPS 服务器
    server {
        listen 443 ssl http2;
        server_name your-domain.com;

        # SSL 证书
        ssl_certificate /etc/nginx/ssl/fullchain.pem;
        ssl_certificate_key /etc/nginx/ssl/privkey.pem;

        # SSL 配置
        ssl_protocols TLSv1.2 TLSv1.3;
        ssl_ciphers HIGH:!aNULL:!MD5;
        ssl_prefer_server_ciphers on;
        ssl_session_cache shared:SSL:10m;
        ssl_session_timeout 10m;

        # 安全头
        add_header X-Frame-Options DENY always;
        add_header X-Content-Type-Options nosniff always;
        add_header X-XSS-Protection "1; mode=block" always;
        add_header Referrer-Policy strict-origin-when-cross-origin always;

        # API 代理
        location /api/ {
            proxy_pass http://neochat_api;
            proxy_http_version 1.1;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            
            # WebSocket 支持
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
            
            # 超时设置
            proxy_connect_timeout 60s;
            proxy_send_timeout 60s;
            proxy_read_timeout 300s;
            
            # 缓冲设置
            proxy_buffering off;
            proxy_request_buffering off;
        }

        # 静态文件 - 上传文件
        location /uploads/ {
            alias /var/www/uploads/;
            expires 30d;
            add_header Cache-Control "public, immutable";
        }

        # Swagger UI
        location /swagger/ {
            proxy_pass http://neochat_api;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }

        # 健康检查
        location /health {
            proxy_pass http://neochat_api;
            access_log off;
        }
    }
}
```

### 获取 SSL 证书 (Let's Encrypt)

```bash
# 安装 Certbot
sudo apt install certbot -y

# 获取证书
sudo certbot certonly --standalone -d your-domain.com

# 证书将保存在:
# /etc/letsencrypt/live/your-domain.com/fullchain.pem
# /etc/letsencrypt/live/your-domain.com/privkey.pem

# 复制证书到 nginx 目录
sudo mkdir -p /etc/nginx/ssl
sudo cp /etc/letsencrypt/live/your-domain.com/fullchain.pem /etc/nginx/ssl/
sudo cp /etc/letsencrypt/live/your-domain.com/privkey.pem /etc/nginx/ssl/

# 设置自动续期
sudo crontab -e
# 添加: 0 3 * * * certbot renew --quiet --post-hook "systemctl reload nginx"
```

## 数据库配置

### PostgreSQL 备份

#### 自动备份脚本

创建 `/opt/neochat/scripts/backup.sh`:

```bash
#!/bin/bash

# 配置
BACKUP_DIR="/opt/neochat/backups"
DATE=$(date +%Y%m%d_%H%M%S)
DB_NAME="neochat"
DB_USER="neochat"
RETENTION_DAYS=7

# 创建备份目录
mkdir -p "$BACKUP_DIR"

# 备份数据库
pg_dump -U "$DB_USER" "$DB_NAME" | gzip > "$BACKUP_DIR/neochat_$DATE.sql.gz"

# 删除旧备份
find "$BACKUP_DIR" -name "neochat_*.sql.gz" -mtime +$RETENTION_DAYS -delete

echo "Backup completed: neochat_$DATE.sql.gz"
```

#### 设置定时任务

```bash
# 编辑 crontab
sudo crontab -e

# 每天凌晨 2 点备份
0 2 * * * /opt/neochat/scripts/backup.sh >> /var/log/neochat-backup.log 2>&1
```

### PostgreSQL 恢复

```bash
# 停止服务
sudo systemctl stop neochat-api

# 删除现有数据库
sudo -u postgres dropdb neochat

# 创建新数据库
sudo -u postgres createdb -O neochat neochat

# 恢复备份
gunzip -c /path/to/backup.sql.gz | sudo -u postgres psql neochat

# 启动服务
sudo systemctl start neochat-api
```

## 故障排查

### 服务无法启动

```bash
# 查看服务状态
sudo systemctl status neochat-api

# 查看详细日志
sudo journalctl -u neochat-api -n 50 --no-pager

# 检查配置文件语法
cd /opt/neochat
./neochat-api -config-check
```

### 数据库连接失败

```bash
# 检查 PostgreSQL 状态
sudo systemctl status postgresql

# 测试连接
sudo -u postgres psql -d neochat -c "SELECT 1;"

# 检查 PostgreSQL 日志
sudo tail -f /var/log/postgresql/postgresql-15-main.log
```

### Redis 连接失败

```bash
# 检查 Redis 状态
sudo systemctl status redis

# 测试连接
redis-cli ping

# 检查 Redis 日志
sudo tail -f /var/log/redis/redis-server.log
```

### 端口被占用

```bash
# 查看端口占用
sudo netstat -tlnp | grep 8080
# 或
sudo lsof -i :8080

# 杀死占用进程
sudo kill -9 <PID>
```

### 权限问题

```bash
# 修复权限
sudo chown -R neochat:neochat /opt/neochat
sudo chmod -R 750 /opt/neochat
sudo chmod -R 770 /opt/neochat/uploads
```

## 安全建议

1. **定期更新系统和软件包**
   ```bash
   sudo apt update && sudo apt upgrade -y
   ```

2. **配置防火墙**
   ```bash
   sudo ufw allow 22/tcp
   sudo ufw allow 80/tcp
   sudo ufw allow 443/tcp
   sudo ufw enable
   ```

3. **禁用 root 登录**
   ```bash
   # 编辑 /etc/ssh/sshd_config
   PermitRootLogin no
   
   # 重启 SSH
   sudo systemctl restart sshd
   ```

4. **使用密钥认证**
   ```bash
   # 在本地生成密钥
   ssh-keygen -t ed25519
   
   # 复制公钥到服务器
   ssh-copy-id user@your-server
   ```

5. **定期备份**
   - 确保备份脚本正常运行
   - 定期测试恢复流程
   - 将备份存储在异地

6. **监控日志**
   - 定期检查异常登录
   - 关注 API 错误率
   - 监控系统资源使用
