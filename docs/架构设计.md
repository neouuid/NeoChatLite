## 技术选型

### 客户端技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **框架** | React Native 0.73+ | 跨平台移动应用框架 |
| **桌面端** | React Native for Windows + macOS | 微软官方维护 |
| **语言** | TypeScript 5.3+ | 类型安全 |
| **状态管理** | Zustand | 轻量级，简单易用 |

### 数据流向

#### 1. HTTP 请求流程

```
客户端 → API Gateway → 认证中间件 → 对应 Service → 数据库 → 响应
```

#### 2. WebSocket 实时消息流程

```
发送方 → WebSocket Gateway → NATS JetStream → 接收方的 WebSocket 连接 → 接收方
                        ↓
                   持久化到 PostgreSQL
```

#### 3. 音视频通话流程

```
呼叫方 → RTC Signaling Service → 信令交换 → WebRTC P2P 连接 → 通话
                 ↓
            保存通话记录
```

### 部署架构说明

| 组件 | 部署方式 | 说明 |
|------|----------|------|
| API Gateway | Kubernetes Deployment + HPA | 无状态，可水平扩展 |
| Auth Service | Kubernetes Deployment + HPA | 无状态，可水平扩展 |
| Chat Service | Kubernetes Deployment + HPA | 无状态，可水平扩展 |

| MinIO / OSS | 对象存储服务 | 高可用 |

## 技术选型详细对比

### React Native vs Flutter

#### 选型决策: React Native

| 维度 | React Native | Flutter | 理由 |
|------|-------------|---------|------|
| WebRTC 生态 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | react-native-webrtc 更成熟 |
| 桌面端支持 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | RNW 由微软官方维护 |
| 代码复用 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 两者都很好 |
| 性能 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | Flutter 略优，但聊天够用 |
| 生态系统 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | RN 第三方库更丰富 |
| 学习曲线 | ⭐⭐⭐⭐ | ⭐⭐⭐ | 有 React 基础选 RN |
| 热重载 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | Flutter 稍好 |
| 包体积 | ⭐⭐⭐ | ⭐⭐ | RN 略小 |

#### 关键决定因素

1. **WebRTC 成熟度**: react-native-webrtc 在生产环境中有大量验证
2. **桌面端**: React Native for Windows/macOS 由微软官方维护，更稳定
3. **团队技能**: 与 Web 技术栈衔接更好

| 学习曲线 | ⭐⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐ |
| 样板代码 | 极少 | 多 | 中等 |

| 性能 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| DevTools | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| 社区规模 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |

#### 关键决定因素

- 简单易用，API 简洁
- TypeScript 支持优秀
- 不需要太多样板代码
- 足够满足聊天应用的需求

## 安全架构

### 认证安全

- **JWT**: 无状态认证，支持刷新
- **密码**: bcrypt 哈希存储
- **Token**: Access Token + Refresh Token 双 Token 机制
- **HTTPS**: 强制加密传输
- **CORS**: 严格的跨域策略

### 数据安全

- **数据库加密**: 静态数据加密
- **传输加密**: TLS 1.3
- **敏感数据**: 手机号、邮箱脱敏存储
- **权限控制**: 基于角色的访问控制 (RBAC)

### API 安全

- **速率限制**: 防止暴力攻击
- **输入验证**: 严格的参数验证
- **SQL 注入防护**: ORM 参数化查询
- **XSS 防护**: 输出编码
