## 概述

NeoChat 使用以下测试框架和工具：

- **后端 (Go)**:
  - 标准库 `testing` 包
  - `testify` 用于增强断言和 mock
  - 表驱动测试模式

- **前端 (TypeScript/React Native)**:
  - `Jest` 作为测试运行器
  - `@testing-library/react-native` 用于组件测试
  - `@testing-library/react-hooks` 用于 Hook 测试

## 后端测试

### 运行测试

```bash
cd backend

# 运行所有测试
go test ./...

# 运行特定包的测试
go test ./pkg/utils -v

# 运行单个测试
go test ./pkg/utils -run TestHashPassword -v

# 查看测试覆盖率
go test ./... -coverprofile=coverage.out
go tool cover -html=coverage.out
```

### 测试文件命名规范

- 测试文件命名为 `*_test.go`，与被测试的文件放在同一目录
- 例如：`password.go` 的测试文件为 `password_test.go`

### 测试工具 (pkg/test)

项目提供了测试工具包 `pkg/test`，包含：

- `SetupTestConfig()` - 设置测试配置
- `SetupTestDatabase()` - 设置 SQLite 内存测试数据库
- `SetupTestRouter()` - 设置测试 Gin 路由
- `CreateTestUser()` - 创建测试用户
- `GetTestToken()` - 获取测试 JWT token
- `MakeRequest()` - 执行 HTTP 测试请求
- `ParseResponse()` - 解析 HTTP 响应
- `AssertStatus()` - 断言 HTTP 状态码
- `SkipIntegrationTests()` - 跳过集成测试的辅助函数
- `StringPtr()`, `IntPtr()` - 指针辅助函数

### 测试示例

#### 基础测试

```go
package utils

import (
    "testing"
    "github.com/stretchr/testify/assert"
)

func TestHashPassword(t *testing.T) {
    hash, err := HashPassword("test123")
    assert.NoError(t, err)
    assert.NotEmpty(t, hash)
}
```

#### 表驱动测试

```go
func TestCheckPasswordHash(t *testing.T) {
    tests := []struct {
        name     string
        password string
        hash     string
        want     bool
    }{
        {
            name:     "正确密码",
            password: "test123",
            hash:     hash,
            want:     true,
        },
        {
            name:     "错误密码",
            password: "wrong",
            hash:     hash,
            want:     false,
        },
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            result := CheckPasswordHash(tt.password, tt.hash)
            assert.Equal(t, tt.want, result)
        })
    }
}
```

### 集成测试

#### 概述

集成测试使用 SQLite 内存数据库进行快速、隔离的测试。主要覆盖：
- Auth 模块：用户注册、登录、资料获取等
- Chat 模块：会话管理、消息发送、收藏等
- WebSocket：连接管理和消息广播

#### 运行集成测试

集成测试需要设置环境变量 `RUN_INTEGRATION_TESTS=true`：

```bash
# 运行所有集成测试
RUN_INTEGRATION_TESTS=true go test ./... -v

# 只运行 Auth 模块集成测试
RUN_INTEGRATION_TESTS=true go test ./internal/auth -v

# 只运行 Chat 模块集成测试
RUN_INTEGRATION_TESTS=true go test ./internal/chat -v

# 运行特定集成测试
RUN_INTEGRATION_TESTS=true go test ./internal/auth -run TestAuthIntegration -v
```

#### 集成测试示例

集成测试使用测试工具包快速设置测试环境：

```go
func TestAuthIntegration(t *testing.T) {
    test.SkipIntegrationTests(t)

    // 设置测试环境
    cfg := test.SetupTestConfig(t)
    setup := test.SetupTestRouter(t, cfg)
    defer test.CleanupTestDatabase(t)

    // 创建测试用户
    user := test.CreateTestUser(t, setup.DB, "testuser", "test@example.com", "password123")

    // 获取 token
    token := test.GetTestToken(t, setup.AuthService, user.ID)

    // 测试 API 端点
    w := test.MakeRequest(t, setup.Router, "GET", "/api/v1/auth/profile", nil, token)
    test.AssertStatus(t, w, 200)

    var response map[string]interface{}
    test.ParseResponse(t, w, &response)
    assert.True(t, response["success"].(bool))
}
```

#### 测试数据库

集成测试使用 SQLite 内存数据库，特点：
- 快速：无需真实的 PostgreSQL 实例
- 隔离：每个测试使用独立的数据库
- 自动清理：测试结束后自动清理

配置在 `config.TestConfig()` 中定义。

## 测试最佳实践

### 1. 测试结构

使用 AAA (Arrange-Act-Assert) 模式：

```go
func TestSomething(t *testing.T) {
    // Arrange: 准备测试数据
    input := "test"
    
    // Act: 执行被测试的函数
    result := DoSomething(input)
    
    // Assert: 验证结果
    assert.Equal(t, expected, result)
}
```

### 2. 测试命名

- 使用描述性的测试名称
- 格式：`Test<FunctionName>_<Scenario>`
- 例如：`TestHashPassword_EmptyPassword`

### 3. 测试隔离

- 每个测试应该独立运行
- 测试之间不应该有依赖关系
- 在 `beforeEach` 中重置状态

### 4. Mock 外部依赖

- 测试 repository 层时使用内存数据库
- 测试 service 层时 mock repository
- 测试 API handler 时使用 `httptest`

### 5. 测试覆盖率

- 目标：核心业务逻辑覆盖率 > 80%
- 不需要追求 100% 覆盖率
- 重点测试关键路径和边界条件
